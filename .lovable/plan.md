
# Landing Page Optimization + Google Sign-In

## Overview

Two focused changes:
1. **Google Sign-In** added to the login page as the primary auth method, with email/password below and Patreon moved to a "Legacy Access" collapsible section at the bottom
2. **Landing Page** cleaned up to remove Patreon dependency, make the Store more prominent, consolidate redundant sections, and give non-members a clearer conversion path

---

## Change 1: Google Sign-In on the Login Page

### How It Works
Lovable Cloud has built-in managed Google OAuth — no API keys or configuration needed. The implementation uses the `lovable.auth.signInWithOAuth("google")` function from `@lovable.dev/cloud-auth-js`.

The `supabase--configure-social-auth` tool will be called first to scaffold the Lovable integration module, then the Login page is updated.

### Files Modified

## Change 2: Landing Page Optimization

### Redundancy Audit
The current page has these issues:
- **Hero CTA** says "Start Your Training" but goes to Patreon (`handleJoinPatreon`)
- **"Powered by Patreon"** text directly under hero CTA — positions Patreon as the platform
- **Hero "Already a member? Log in"** calls `signInWithPatreon()` directly — bypasses the new Login page
- **"How It Works" Step 01** says "Join via Patreon"
- **"What This Is"** accordion buttons say "Join on Patreon"
- **Bottom CTA section** has two buttons: "Join on Patreon" + "Log In with Patreon" — both Patreon-only
- **Store is never mentioned** anywhere on the landing page — it's invisible to non-members
- **"Why This Exists"** section has good content but appears after the Sync Quiz and Patreon blog — buried
- **PatreonBlog component** mid-page creates a Patreon-branded section that signals "you need Patreon"

### Specific Changes

**Hero Section:**
- "Start Your Training" button → routes to `/login?tab=signup` (Join Free)
- Remove "Powered by Patreon" sub-text
- "Already a member? Log in" → routes to `/login` 
- Keep "Explore the Lab" anchor link

**"How It Works" Steps:**
- Step 01: Change "Join via Patreon" → "Create your free account" with note about Google, email, or Patreon
- Step 02: Keep as-is
- Step 03: Keep as-is

**"What This Is" Accordion Buttons:**
- Change "Join on Patreon" buttons → "Join Free" linking to `/login?tab=signup`
- Keep "Apply for Creator Lab" buttons

**Store Section (NEW — added between Free Resources and Why This Exists):**
A new section highlighting that the Store is available to everyone — no membership needed. Shows top 2-3 featured products (Catalog Audit, top guides) with prices and a "Visit the Store" CTA.
```
┌─────────────────────────────────────────┐
│  SECTION LABEL: The Store               │
│  "Tools you can buy today."             │
│  "No membership required."             │
│                                         │
│  [Catalog Audit $X]  [Guide $X]  [...] │
│  Brief description + price each         │
│                                         │
│  [Browse the Full Store →]              │
└─────────────────────────────────────────┘
```

**Bottom CTA Section:**
- Replace "Join on Patreon" + "Log In with Patreon" buttons
- New: "Create Free Account" (primary, routes to `/login?tab=signup`) + "Log In" (secondary, routes to `/login`)
- Remove "Explore the Lab" anchor at the bottom (redundant — already in hero)

**PatreonBlog:** Keep but move to after the Store section, rename the label from "From the Blog" to just "Latest Updates" to reduce the Patreon branding feel.

### Section Order (Revised)
```
1. Hero (CTAs now → /login)
2. What This Is (accordion, buttons → /login?tab=signup)
3. How It Works (step 01 de-Patreon'd)
4. Pricing (kept as-is — shows tiers)
5. FREE: Store Section (NEW — prominent, no login required)
6. Free Resources (quiz, guides — kept)
7. Why This Exists (moved up slightly)
8. Sync Readiness Quiz embed
9. Latest Updates (Patreon blog, de-branded label)
10. Events
11. Partners & Sponsors
12. Bottom CTA (now routes to /login, not Patreon)
```

## Technical Notes

- Google OAuth uses Lovable's managed credentials — no API key setup required from the user
- The `configure-social-auth` tool generates the `src/integrations/lovable/` module automatically
- Google sign-in will trigger the same `handle_new_user` database trigger already in place, so new Google users automatically get a profile with `patreon_tier = 'lab-pass'`
- Existing Patreon OAuth flow is fully preserved — just moved to a "Legacy Access" collapsible on the login page
- All landing page changes are frontend-only — no database changes needed
